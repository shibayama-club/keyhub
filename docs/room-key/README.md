## 概要

この機能により、テナントは実際の物理リソース（部屋・鍵）を管理できるようになります。

### ビジネス要件

- テナントには複数の部屋を割り当てることができる
- 各部屋には複数の鍵を関連付けることができる
- 鍵にはステータス（貸出中、返却済み、紛失など）を持たせる
- 部屋の利用状況を追跡できるようにする
- Console管理者がこれらのリソースを一元管理できる

## 主要な設計決定

1. **ドメイン駆動設計**: Value ObjectとEntityで型安全性を確保
2. **マルチテナンシー**: Organization IDでデータを分離
3. **履歴管理**: 割り当ての時系列情報を保持
4. **拡張性**: 将来的な機能追加（貸出履歴、予約システムなど）を考慮
